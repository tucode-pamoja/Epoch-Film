# 02. 상세 기능 명세서 (Functional Specifications)

## 1. 개요

본 문서는 EPOCH FILM 프로젝트의 기능 요구사항을 구체화하여 개발 기준을 수립함을 목적으로 한다.

## 2. 사용자 인증 (User Authentication)

### 2.1 회원가입/로그인

- **소셜 로그인:** Google, Kakao, Apple 로그인 지원 (NextAuth.js / Supabase Auth)
- **추가 정보 입력:**
  - 닉네임 (중복 확인)
  - MBTI (선택 사항)
  - 선호 카테고리 (최소 1개 이상 선택)
- **정책 동의:** 이용약관, 개인정보처리방침 동의 (필수), 마케팅 수신 동의 (선택)

### 2.2 회원 관리

- **프로필 수정:** 닉네임, 프로필 사진, 한 줄 소개
- **계정 탈퇴:** 7일 유예 기간 후 데이터 영구 삭제 (Soft Delete 우선 적용)

## 3. 현상소 (The Lab) - 버킷리스트 관리

### 3.1 통합 저장소 (Archive)

- **리스트 조회:**
  - 필터: 전체 / 카테고리별 / 상태별 (진행중, 완료)
  - 정렬: 최신순 / 중요도순 / 가나다순
- **리스트 작성 (CRUD):**
  - **입력 필드:** 제목(필수), 상세 내용, 카테고리, 태그, 중요도(1~5)
  - **상태 관리:** `Draft`(작성중) -> `Active`(진행중) -> `Achieved`(완료)
- **검색:** 제목 및 태그 검색

### 3.2 올해의 시퀀스 (Selected Sequence)

- **선정:** Archive에 있는 리스트 중 최대 10개까지 '올해의 목표'로 선정 가능
- **대시보드:** 홈 화면에서 선정된 리스트의 진행률(Progress bar) 표시
- **위젯 연동:** 모바일 홈 위젯에 우선 노출

## 4. 컷 & 셔터 (Cut & Shutter) - 기록 시스템

### 4.1 체크인 샷 (Check-in Shot)

- **기능:** 버킷리스트 완료 시 '체크인' 버튼 활성화
- **미디어:**
  - 사진: 1장 이상 필수 (최대 10장)
  - 영상: 1분 이내 숏폼 (선택)
- **메타데이터:**
  - 촬영 날짜/시간 자동 기록 (Exif 기반)
  - GPS 위치 정보 자동 기록 (사용자 동의 시)
  - 수동 보정 가능 (날짜/위치 수정 허용)

### 4.2 디지털 LP 아카이브

- **LP 생성:** 완료된 버킷은 하나의 '트랙'으로 생성
- **시각화:** LP판이 돌아가는 애니메이션과 함께 선택한 BGM 재생 (YouTube Music / Spotify 연동 고려)
- **커버 아트:** 체크인 샷 중 대표 이미지를 LP 커버로 사용

### 4.3 타임 캡슐 편지 (Letter to Future)

- **작성:** 버킷 달성 시점에 '1년 뒤 나에게' 편지 작성
- **알림:** 설정된 날짜에 앱 푸시 및 이메일 발송
- **잠금:** 오픈 날짜 전까지 열람 불가 (D-Day 카운트다운 표시)

## 5. AI Director (Premium)

### 5.1 로드맵 생성

- **Input:** 버킷리스트 제목 (예: "파리에서 에펠탑 보며 와인 마시기")
- **Process:** LLM이 목표 분석 -> 실행 단계 분해 -> 비용 추산 -> 일정 제안
- **Output:** 단계별 To-Do List, 예상 예산, 추천 시기

### 5.2 스마트 추천

- **위치 기반:** 현재 위치 주변의 '내 버킷리스트' 관련 스팟 알림

## 6. 시사회 (Premiere) - 커뮤니티

### 6.1 명예의 전당 (Hall of Fame)

- **배지 획득:** 특정 조건 달성 시 배지 부여 (예: 여행 카테고리 10개 달성 -> '방랑자' 배지)
- **전시:** 획득한 배지를 3D 오브젝트 형태로 회전/확대 축소하며 감상

### 6.2 캐스팅 (Casting)

- **매칭:** 동일한 버킷리스트를 가진 사용자 추천 (익명 기반)
- **응원하기:** '좋아요' 대신 'Flash'(조명) 기능으로 응원

## 7. 데이터 관리 및 보안

- **백업:** 매일 1회 DB 백업
- **이미지 최적화:** 업로드 시 WEBP 포맷 변환 및 리사이징 (Lambda@Edge 등 활용)
